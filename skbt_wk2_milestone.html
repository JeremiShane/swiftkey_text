Executive Summary

The following report demonstrates familiarity and facility with the three English language text data files for the Swiftkey capstone project for Johns Hopkins Data Scinece certification on Coursera. The purpose of the project is to publish a shiny app that will predict the next word as users type in text.

    The three files are:
    1. blogs
    2. twitter
    3. news

The following milestone report shows basic exploration of the files and the terms. Next steps are to further scrub the terms, evaluate other data sources to add to the corpus, and build a Naive Bayes predictive model that will predict the next word to be typed based on the last one, two or three words typed.

Exploratory Analysis of the Files

frequencies of words and word pairs

During this exploratory and model building phase, only a sample (100000 lines from each) of the files will be used for computational efficiency.
More data will be included for the final training set.

After reading in the files, I first remove all non-ascii characters. After converting the files to a corpus, I use the tm package to cleanse the data then remove all infrequent terms which I have defined as occuring less than 13 times.

Using 13 as the threshold I determined that ~15% of the total terms will remain in the dataset. This is a dramatic reduction in the size of the corpus, and with the most relavent, predictive and predictable terms remaining.

Next Steps: 1. to stem or not to stem final training data 2. decide on additional data to include 3. build Naive Bayes predictive model 4. build application for predicting next word and post to web
